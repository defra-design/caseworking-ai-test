Contributing

How to use, extend and maintain this library so the prototype stays consistent as it grows.

The order of preference

Always reach for the most established option first:

  1. GOV.UK Design System — use a GDS component if one fits.
  2. MOJ Design System — for internal casework needs GDS doesn't cover (timelines, filters, sortable tables).
  3. Custom — only when neither system has what you need, and only after checking with the team.

A new custom pattern is a cost: it must be documented here, kept accessible, and maintained. Don't create one to save a few minutes.

Proposing a pattern

  1. Check it doesn't already exist in GDS, MOJ or here.
  2. Build it from tokens (colour, spacing) — no magic numbers, no raw hex.
  3. Document it: copy _TEMPLATE.html and fill every section, including accessibility and "where does this occur".
  4. Add it to the relevant subnav include and the landing page count.
  5. Give it a status label (below).

Component status

Each page shows a status tag in its heading so users know how much to trust it:

  • Ready   Documented, accessible, used in the prototype.
  • Experimental   In progress or used once — may change.
  • Deprecated   Don't use in new work; being removed.

Naming & code conventions

  • Tokens are prefixed $cw- and live in _tokens.scss.
  • App layout classes are prefixed .cw-; pattern-library chrome is prefixed .app-.
  • Keep using the GOV.UK and MOJ class names for ported components so they can be swapped for the real packages.
  • Escape HTML inside example code blocks (&lt; &gt;).

Changing or retiring a pattern

  • Changing a token or component affects every screen — search usages first and test them.
  • To retire one, mark it Deprecated, migrate usages, then remove it and its subnav entry.
  • Record notable changes in the "What's new" list on the landing page.

Housekeeping backlog

Tracked in MIGRATION.md and the upgrade package's CLAUDE.md:

  • Replace magic-number classes with tokens, then delete the legacy shims.
  • Consolidate the overlapping tag variants.
  • Split application.scss into partials.
  • Decide whether to install MOJ Frontend or formally adopt the ports.
  • Finish the missing component and pattern pages.
