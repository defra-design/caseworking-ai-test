Focus & interaction states

Interactive elements must show a clear focus state for keyboard users. The service uses the GOV.UK focus style: a yellow background with a 3px black underline.

The focus state

Example

Tab to the link to see the focus state, or preview it here:

Focused link

// _tokens.scss
$cw-focus-bg:    #ffdd00; // govuk-colour('yellow')
$cw-focus-text:  #0b0c0c;
$cw-focus-width: 3px;

// GOV.UK focus mixin (preferred): @include govuk-focused-text;

Buttons and links

Example 
<button class="govuk-button" data-module="govuk-button">Primary</button>
<button class="govuk-button govuk-button--secondary">Secondary</button>
<button class="govuk-button govuk-button--warning">Warning</button>

Rules

  • Never remove focus outlines with outline: none without providing an equally visible replacement.
  • Custom controls (the quiet secondary button, filter tags, sortable headers) must show the same yellow focus state — see the Accessibility page.
  • Hover is a supplementary cue only; never rely on hover to convey state.
  • Hit targets are at least 44 × 44px.

Where does this occur

All interactive components. Watch the custom .gem-c-button--secondary-quiet and the filter tags, which were styled by hand and must keep the standard focus state.
