Colour

Every colour in this service is a named design token defined in app/assets/sass/_tokens.scss. Use the token, never a raw hex or a compiled value.

Colours used to be written as raw hex or, worse, as compiled values such as rgb(42.4, 10.6, 5.6). Those are now the tokens $cw-tag-red-text and $cw-tag-red-bg. If you need a colour that isn't here, add a token — don't inline a value.

Brand

Defra green is the brand colour. The darker green is used on the pattern-library header and on green link hover states.

Example
Swatch Token Value Use
$cw-brand-green #00a33b Defra brand
$cw-brand-green-pl #008938 Pattern-library header
$cw-brand-green-dark #005a25 Green link hover 
// _tokens.scss
$cw-brand-green:      #00a33b;
$cw-brand-green-pl:   #008938;
$cw-brand-green-dark: #005a25;

// Usage in Sass
.thing { border-bottom: 8px solid $cw-brand-green; }

Text, border and surface

These map to the GOV.UK greys. Body text is near-black; secondary text is dark grey; borders are mid grey; muted surfaces are light grey.

Example
Swatch Token Value Use
$cw-text #0b0c0c Body text
$cw-text-secondary #505a5f Secondary / hint text
$cw-border #b1b4b6 Borders, rules
$cw-surface-muted #f3f2f1 Muted surfaces, code blocks
$cw-surface-success #e7f4eb Pale green band (.cw-panel-success) 
$cw-text:            #0b0c0c;
$cw-text-secondary:  #505a5f;
$cw-border:          #b1b4b6;
$cw-surface-muted:   #f3f2f1;
$cw-surface-success: #e7f4eb;

Status colour

Status colours carry meaning. Map decisions to colours consistently — the full mapping lives on the Status tags page. The two custom tints below — the soft red govuk-tag--red-status ("Not accepted") and the pale blue govuk-tag--action ("Action required") — extend the GOV.UK tag palette.

Example

Accepted   In review   Information requested

Rejected   Withdrawn

Not accepted   Incomplete   Action required

// Custom tints (_tokens.scss → _tags.scss)
$cw-tag-red-text:    #2a0b06;  // was rgb(42.4, 10.6, 5.6)
$cw-tag-red-bg:      #f4ccc6;  // was rgb(244.25, 204.5, 198.25)
$cw-tag-action-text: #0f385c;
$cw-tag-action-bg:   #d2e2f1;

<strong class="govuk-tag govuk-tag--red-status">Not accepted</strong>
<strong class="govuk-tag govuk-tag--neutral">Incomplete</strong>
<strong class="govuk-tag govuk-tag--action">Action required</strong>

Accessibility

  • Never use colour as the only way to convey meaning — always pair a status colour with text (the tag label does this).
  • Body text on white meets WCAG AA. Check any custom tint you introduce reaches 4.5:1 against its background; note results on the Accessibility page.
  • The focus state uses $cw-yellow (#ffdd00) with a black underline — see Focus & interaction states.

Where does this occur

Colour tokens underpin every screen. The status tints appear most visibly in the caselist, the task list and the calculations pages.
