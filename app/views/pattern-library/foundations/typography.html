Typography

The service uses the GOV.UK type system: the GDS Transport typeface and the GOV.UK type scale. Use the GOV.UK heading and body classes — don't set font sizes by hand.

Font family token: $cw-font = "GDS Transport", arial, sans-serif. Several places in the old CSS re-declared this stack inline (for example .small-text, .govuk-heading-s-w). Prefer the GOV.UK classes below.

Headings

Example

Heading XL — page title

Heading L — section

Heading M — subsection

Heading S — small heading

<h1 class="govuk-heading-xl">Page title</h1>
<h2 class="govuk-heading-l">Section</h2>
<h3 class="govuk-heading-m">Subsection</h3>
<h4 class="govuk-heading-s">Small heading</h4>

Body and supporting text

Example

Body large — lead paragraph that introduces a page.

Body — the default paragraph size for most content on a page.

Body small — use sparingly, for metadata and captions.

<p class="govuk-body-l">Lead paragraph</p>
<p class="govuk-body">Default paragraph</p>
<p class="govuk-body-s">Metadata / caption</p>

The type scale

GOV.UK font sizes are responsive (larger on tablet and above). Use the class, not the pixel value.

Class Desktop size Use
govuk-heading-xl48pxPage title (one per page)
govuk-heading-l36pxSection heading
govuk-heading-m24pxSubsection
govuk-heading-s / govuk-body19pxSmall heading / body
govuk-body-s16pxMetadata, captions

Accessibility

  • Keep one <h1> per page and don't skip heading levels — use the visual class to size, the tag for structure.
  • Avoid .small-text (16px override) for primary content; reserve small text for genuine metadata.
  • Don't hard-code font sizes — the responsive scale keeps text legible across breakpoints.

Where does this occur

Every page. The heading hierarchy is most load-bearing on the case task list (the tasklist-stage screen) and the calculations pattern.
