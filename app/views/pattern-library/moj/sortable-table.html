Sortable table

Let users sort a table of cases or records by clicking any column heading.

Use the MOJ sortable table to make long lists of cases easier to scan. Each sortable column heading is a button — clicking it reorders the rows by that column and toggles between ascending and descending order. The component is progressive: rows remain in their original order if JavaScript is unavailable.

When to use

  • For caselists, worklists or any tabular list users need to reorder.
  • When sorting on the server would be too slow or unnecessary for the data set size.
  • When all the data the user will sort is already on the page.

When not to use

  • If the table is paginated server-side — sorting client-side would only reorder the visible page. Use server-driven sort links instead.
  • For data that needs sorting on derived values the cell doesn’t show; supply a data-sort-value attribute on those cells.

Initial sort state

Set aria-sort="ascending" or aria-sort="descending" on the column heading the table should be sorted by when it first loads. Set aria-sort="none" on every other sortable column.
