Accessible autocomplete

An accessible, type-ahead select input from alphagov that lets a user filter a long list of options by typing.

It is published as a separate package (accessible-autocomplete) and is not part of govuk-frontend itself. Use it whenever the user needs to pick one value from a long, known list — typically too long for a standard <select> or a radio group to remain usable.

Default autocomplete

Example 
<link rel="stylesheet" href="/vendor/accessible-autocomplete/accessible-autocomplete.min.css">

<div class="govuk-form-group">
  <label class="govuk-label govuk-label--s" for="pl-aa-input">
    Find a specific land parcel
  </label>
  <div id="pl-aa-container"></div>
</div>

<script src="/vendor/accessible-autocomplete/accessible-autocomplete.min.js"></script>
<script>
  accessibleAutocomplete({
    element: document.getElementById('pl-aa-container'),
    id: 'pl-aa-input',
    source: ['SD5848 9205', 'SD5649 9215', 'NY2105 1006', '...'],
    minLength: 1,
    confirmOnBlur: false,
    displayMenu: 'overlay',
    placeholder: 'e.g. SD5848 9205'
  });
</script>

When to use

  • The user needs to pick one item from a long, finite list (more than ~20 items) where they probably already know what they want.
  • The list is known up front and small enough to ship to the browser (one-page calculations with up to a few hundred land parcels, for example).
  • You want suggestions as the user types, with full keyboard and screen-reader support out of the box.

When not to use

  • For short lists (≤ 10 items) — use radios or a standard <select>.
  • For free-text input where the answer might not match the list — use a plain <input> instead.
  • For server-paginated options that cannot all be sent to the page at once — use a paginated search pattern.

Customisation in this prototype

The autocomplete input and its suggestion menu inherit the GDS Transport font via a small SCSS override on .autocomplete__input, .autocomplete__menu and .autocomplete__option in application.scss — the upstream stylesheet defaults to a generic sans-serif.

Where does this occur

Used in the parcel-search panel above the parcel list on every large calculations page (paired with a Find button that hides every other parcel block in place when a parcel is selected).
