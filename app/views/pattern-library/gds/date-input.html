Date input Ready

Use the date input to ask for a memorable date or one the user can look up — three separate fields for day, month and year.

When to use this

  • Asking for a date a user knows (e.g. date received, date of an inspection).

When not to use this

  • Choosing from a small set of known dates — use radios or a select.
  • Don't use a native date picker — the three-field pattern is more accessible for known dates.

Example

Example
Date received
For example, 27 3 2026
<fieldset class="govuk-fieldset" role="group" aria-describedby="received-hint">
  <legend class="govuk-fieldset__legend govuk-fieldset__legend--m">Date received</legend>
  <div id="received-hint" class="govuk-hint">For example, 27 3 2026</div>
  <div class="govuk-date-input" id="received">
    <div class="govuk-date-input__item">
      <div class="govuk-form-group">
        <label class="govuk-label govuk-date-input__label" for="received-day">Day</label>
        <input class="govuk-input govuk-date-input__input govuk-input--width-2" id="received-day" name="received-day" type="text" inputmode="numeric">
      </div>
    </div>
    <!-- month (width-2), year (width-4) -->
  </div>
</fieldset>

Accessibility

  • Wrap the three fields in a <fieldset role="group"> with a <legend>.
  • Use inputmode="numeric" so mobile users get a numeric keypad, but keep type="text" so values aren't silently changed.
  • On error, highlight the specific field(s) and link from the error summary to the first one.

Where does this occur

The User Management screens in FRPS-D2:

  • Create role
  • Role detail (read only)
