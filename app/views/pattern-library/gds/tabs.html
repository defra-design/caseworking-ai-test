Tabs

Let users switch between related sections of content on the same page without losing the current context.

Use tabs when several views of the same underlying data are useful (for example, the case list filtered by team vs. filtered by me) and the user is likely to switch between them frequently. Each tab loads its own panel; the URL stays the same.

When to use

  • To switch between multiple views of the same underlying content on a single page (e.g. team vs. personal case list).
  • To group related content into two to five sections where the user is likely to want all of them at hand.

When not to use

  • To split a long-form task into steps — use a separate page per step instead.
  • When users need to compare content across panels — keep it on the same page without tabs.
  • When there are more than five panels — consider a navigation menu or a search/filter instead.

Where does this occur

  • Case list — single-tab variant used as a section header (SFI applications (20)).
  • My case list — second tab panel preview.
  • Notes and Add a note — content split between Notes and Messages panels.
