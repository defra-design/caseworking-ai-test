Textarea

A multi-line text input for free-text answers — review notes, messages to customers, justification text.

In this prototype the textarea appears wherever a caseworker needs to leave a written record of a decision. Most uses pair the field with a label and a hint, and live behind a conditional that only renders the form for one branch of a radio set (e.g. only show "Why are you requesting more information?" when the Information requested radio is selected).

When to use

  • For free-text answers longer than a single line — review notes, justifications, messages to customers.
  • When the expected length is variable and the answer cannot be reduced to a select / radio / checkbox.

When not to use

  • For short, single-line answers (one phrase, a name, a reference) — use a text input.
  • When the user is picking from a known list — use radios, checkboxes, a select, or the accessible autocomplete.

Conventions in this prototype

  • rows="5" by default — about five lines of visible space.
  • Always paired with a labelled .govuk-form-group wrapper plus a .govuk-hint when the purpose isn't obvious from the label.
  • Conditionally rendered branch of a radio group — when revealed, the textarea name varies by branch so each captured note is stored under its own session key (for example task1Note2<Case> vs task1Note<Case>).

Where does this occur

In every review-decision form in the case journeys:

  • Task pages: case2/task-1.html through task-6.html and their -Ag, -Am, -Tr variants.
  • 5-month-check tasks: case2/task-5m1.html through task-5m6.html.
  • Assignment, add-a-note, return/amend/terminate confirmation dialogs.
  • Same files mirrored under caseReal/.
