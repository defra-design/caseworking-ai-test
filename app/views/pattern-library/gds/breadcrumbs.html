Breadcrumbs Ready

Breadcrumbs help users understand where they are in a hierarchy and move back up a level. This library uses them on every doc page.

When to use this

  • The service has a clear hierarchy and the user may be more than one level deep.
  • Users arrive part-way through (e.g. from search or a deep link) and need orientation.

When not to use this

  • Linear flows (task list → task → check answers) — use the back link instead.
  • Flat services with no hierarchy.

Example

Example
  1. Cases
  2. 100300 — Berkshire Estate Limited
  3. Documents
<nav class="govuk-breadcrumbs" aria-label="Breadcrumb">
  <ol class="govuk-breadcrumbs__list">
    <li class="govuk-breadcrumbs__list-item"><a class="govuk-breadcrumbs__link" href="#">Cases</a></li>
    <li class="govuk-breadcrumbs__list-item"><a class="govuk-breadcrumbs__link" href="#">100300 — Berkshire Estate Limited</a></li>
    <li class="govuk-breadcrumbs__list-item" aria-current="page">Documents</li>
  </ol>
</nav>

Accessibility

  • Wrap in a <nav aria-label="Breadcrumb">.
  • The current page is the last item, with no link, and aria-current="page".

Where does this occur

Every pattern-library page, and the case sub-pages where a hierarchy exists.
