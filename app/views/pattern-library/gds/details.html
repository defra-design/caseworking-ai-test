Details

Make a page easier to scan by letting users reveal more detailed information only if they need it.

Use the details component to hide secondary content (for example help text, supporting policy notes or rarely-needed information) behind a disclosure link. It keeps the primary content on the page short while still making detail available without a separate page.

Default details

Example
Help with this calculation

The required area check compares the area the customer has applied for with the total area available for the action on this land parcel. If the figures do not match, the check will fail and the case will need to be reviewed before the calculation can be approved.

When to use

  • To reveal help text or a fuller explanation only when the user asks for it.
  • To collapse a long list of supporting metadata into a single, scannable line.
  • To group rarely-used controls (for example a debug panel) out of the main flow.

When not to use

  • To hide content that the user must read to complete the task — keep that on the page.
  • To hide form fields the user is expected to fill in.

Where does this occur

  • Calculations (new)
