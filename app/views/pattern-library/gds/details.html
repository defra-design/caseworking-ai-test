Details

Make a page easier to scan by letting users reveal more detailed information only if they need it.

Use the details component to hide secondary content (for example help text, supporting policy notes or rarely-needed information) behind a disclosure link. It keeps the primary content on the page short while still making detail available without a separate page.

In this prototype the details component does most of its work on the calculations page, where every individual rules check is a details disclosure with its outcome shown as a tag in the summary — see Details as a check summary below.

Default details

Example
Help with this calculation

The required area check compares the area the customer has applied for with the total area available for the action on this land parcel. If the figures do not match, the check will fail and the case will need to be reviewed before the calculation can be approved.

<details class="govuk-details">
  <summary class="govuk-details__summary">
    <span class="govuk-details__summary-text">
      Help with this calculation
    </span>
  </summary>
  <div class="govuk-details__text">
    <p class="govuk-body">The required area check compares the area the
    customer has applied for with the total area available for the action
    on this land parcel.</p>
  </div>
</details>

Details as a check summary

On the calculations pages, the disclosure isn't hiding help text — it's hiding the evidence behind a single rules check. The summary line carries two things: the check name on the left, and a check-result tag (Passed / Failed / Checked / Action) on the right. The caseworker can read every outcome down the page without opening anything, and open only the checks they need to investigate.

Example
Required area check Failed

Expanded content: the result, the fail reason, and the "Check details" table.

Area on moorland check Passed

Expanded content: the result, the pass reason, and the "Check details" table.

<details class="govuk-details govuk-!-margin-bottom-3">
  <summary class="govuk-details__summary">
    <span class="govuk-details__summary-text">
      Required area check
      <strong class="govuk-tag govuk-tag--red-new">Failed</strong>
    </span>
  </summary>
  <div class="govuk-details__text">
    <!-- Result, reason, and Check details table -->
  </div>
</details>

When to use

  • To reveal help text or a fuller explanation only when the user asks for it.
  • To collapse a long list of supporting metadata into a single, scannable line.
  • To present a long, repeating list of results — like rules checks — where each item's outcome can be shown in the summary and its evidence hidden until needed.
  • To group rarely-used controls (for example a debug panel) out of the main flow.

When not to use

  • To hide content that the user must read to complete the task — keep that on the page.
  • To hide form fields the user is expected to fill in.
  • To carry the overall case status in the summary — that's a status tag on the case, not a check-result tag.

Where does this occur

  • Calculations pattern — the per-check disclosure
  • Land parcel calculations
  • Real-case calculations (Golden Grange)
