Text input

A single-line text field for free-text answers that fit on one line.

Used throughout the prototype for short identifiers (case IDs, SBI numbers), search queries, and any one-line input. Apply a width modifier (govuk-input--width-2 through --width-30) when the expected answer is short and a full-width field would be misleading.

Default text input

Width modifiers

Use a width modifier when the answer has a known short length. Sizes are nominal — they assume an unencoded character count.

govuk-input--width-20 — SBI numbers, postcodes

govuk-input--width-10 — case IDs

govuk-input--width-5 — reference codes

Search variant

When the input is the primary search field on the page, pair it with the MOJ search pattern: type="search" and the moj-search__input modifier.

You can search by case ID or SBI.

When to use

  • For short, single-line answers — case IDs, SBI numbers, names, references.
  • For search boxes (paired with the MOJ search styling and a Search button).

When not to use

  • For multi-line free text — use a textarea.
  • For picking from a known list — use radios, checkboxes, a select, or the accessible autocomplete.

Where does this occur

  • Case list search (search variant).
  • Assignment search.
  • My case list filters — also used inline for the input fields inside the filter panel.
