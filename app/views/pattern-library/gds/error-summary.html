Error summary Ready

When a user submits a form with errors, show an error summary at the top of the page that lists every error and links to the field that needs fixing.

When to use this

  • Always, when a form submission fails validation — even if there is only one error.

How it works

  • The summary sits directly below the page <h1>.
  • Each item links to the input it relates to (matching the input's id).
  • Each field also shows an inline error message, and focus moves to the summary on submit.

Example

Example

There is a problem

  • Select a decision
  • Enter a reason for the decision

Error: Enter a reason for the decision

<div class="govuk-error-summary" data-module="govuk-error-summary">
  <div role="alert">
    <h2 class="govuk-error-summary__title">There is a problem</h2>
    <div class="govuk-error-summary__body">
      <ul class="govuk-list govuk-error-summary__list">
        <li><a href="#decision">Select a decision</a></li>
        <li><a href="#reason">Enter a reason for the decision</a></li>
      </ul>
    </div>
  </div>
</div>

<div class="govuk-form-group govuk-form-group--error">
  <label class="govuk-label" for="reason">Reason for the decision</label>
  <p id="reason-error" class="govuk-error-message"><span class="govuk-visually-hidden">Error:</span> Enter a reason</p>
  <input class="govuk-input govuk-input--error" id="reason" aria-describedby="reason-error">
</div>

Accessibility

  • The summary uses role="alert" and receives focus on submit so it's announced.
  • Each summary link moves focus to the related field.
  • Error messages are linked to inputs with aria-describedby and prefixed with a visually-hidden "Error:".
  • Set the page <title> to start with "Error:" when the page has errors.

Where does this occur

Any form in the service: the assignment, amend and return-confirmation screens, and any check that captures a reason.
