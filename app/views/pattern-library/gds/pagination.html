Pagination

Let users navigate a list of results split across multiple pages.

This is the GOV.UK Design System pagination — distinct from the MOJ pagination pattern, which has a slightly different visual style and is used on MOJ list pages. The case list in this prototype uses the GDS variant.

Default pagination

Example
Previous page
  • 1
  • 2
  • 3
Next page

      <!-- prev icon svg -->
      <span class="govuk-pagination__link-title">
        Previous<span class="govuk-visually-hidden"> page</span>
      </span>

GDS vs MOJ pagination

Use the GDS pagination for public-facing or service-style list pages. Use the MOJ pagination for internal casework list pages that also use the MOJ filter and sortable-table components — the visual style sits more naturally alongside the rest of the MOJ component set.

When to use

  • To split a long list of results across multiple pages.
  • When the user needs to jump to a specific page rather than scroll continuously.

When not to use

  • For lists short enough to fit on a single page — pagination adds friction.
  • When the entire list belongs on a single page for cross-comparison (e.g. the parcels in a calculations run).
  • When MOJ filter / sortable-table is already on the page — use the MOJ variant for visual consistency.

Where does this occur

  • Case list (the markup is currently commented out — it appears as soon as the result count exceeds one page).
