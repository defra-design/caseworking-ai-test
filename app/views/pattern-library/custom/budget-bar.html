Budget allocation bar

A single horizontal bar that shows how a funding pot is divided — what's approved, what's in the pipeline, and what's still free — with a labelled key beneath it.

Used on the grants dashboards to give an at-a-glance read of a budget's health. The bar is a flex row of coloured segments whose widths are the percentage of the pot each state holds; the last segment flexes to fill whatever is left. Colours reuse the case tag palette — green for approved, blue for pipeline, grey/white for what remains — so the bar reads consistently with the status tags elsewhere in the service.

Portfolio bar (three segments)

The base form used on the grants overview: app-budget-bar__approved (green), app-budget-bar__pipeline (blue) and app-budget-bar__remaining (grey, flexes to fill). Each segment carries its own percentage; the key beneath sits inline, each item led by a colour swatch.

Approved: £380,000 Pipeline: £460,000 Uncommitted remaining: £3,360,000

Scheme bar (split pipeline, outlined, aligned key)

The extended form used on a per-scheme caselist (e.g. Green Tech Innovation Fund). The pipeline is split into Progressing (blue), On hold (yellow tag colour, black text) and Not started (grey tag colour, black text), with Remaining free shown as plain white. A black outline is drawn around the whole chart via app-budget-bar--outlined (not around the white segment). The key uses the app-budget-key--aligned variant: each item is absolutely positioned at its segment's start (inline left:%), with the swatch aligned to that start and the figure stacked under the label.

Customisation in this prototype

  • The bar is a display:flex row, fixed 36px tall, with overflow:hidden and a 2px radius. Segment widths are set inline as percentages; the final segment (__remaining / __free) uses flex:1 1 auto to absorb the balance.
  • Segment colours match the tag palette: approved #00703c, pipeline/progressing #1d70b8, on-hold #fff7bf (yellow tag), not-started #eeefef (grey tag), remaining #b1b4b6, free white. Light segments carry black text for contrast.
  • --outlined draws the black border around the whole chart (box-sizing:border-box so it doesn't grow the bar).
  • The key has two forms: the default inline flex row, and --aligned — a positioned block the same width as the bar, where each item's left:% must equal the cumulative start of its bar segment for the swatches to line up.

When to use

  • To show how one budget or pot is divided across a small set of states (approved / pipeline / remaining, or a scheme's split).
  • At the top of a dashboard or scheme caselist, as a summary the caseworker reads before the detail.

When not to use

  • For more than about five segments — narrow slices can't hold their percentage label and the aligned key items start to crowd.
  • For time-series or comparison across many pots — use a proper chart, not a single stacked bar.
  • The --aligned key is tuned for desktop widths; on very narrow viewports the positioned items can overlap. Prefer the inline key if the bar must work at small sizes.

Where does this occur

  • Grants overview — the portfolio bar (three segments, inline key) under "Aggregated portfolio health".
  • Green Tech Innovation Fund caselist — the scheme bar (split pipeline, outlined, aligned key) under "Realtime funds allocation".
