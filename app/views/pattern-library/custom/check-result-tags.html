Check-result tags

Pass / Fail / Yes / No status tags with tick and cross icons. Used in the summary row of every rule check on the calculations pages.

There are two visual variants — a green tag with a tick (for any positive outcome) and a red tag with a cross (for any negative outcome) — and two label vocabularies on top of those:

  • Pass / Fail — used when the check has a regulatory pass-or-fail outcome (Available area check, Moorland threshold check).
  • Yes / No — used when the check answers a binary question rather than passing/failing (HEFER required?, SSSI present?). Same visuals, different wording.

Pass

Green govuk-tag--green with an inline .tick-icon svg. The svg is included inline so the icon scales and inherits colour cleanly with the tag.

Example
Passed 
<strong class="govuk-tag govuk-tag--green">
  <svg class="tick-icon" width="20" height="20" viewBox="0 0 24 24"
       fill="green" xmlns="http://www.w3.org/2000/svg">
    <path d="M9 16.2L4.8 12L3.4 13.4L9 19L21 7L19.6 5.6L9 16.2Z"/>
  </svg>
  Passed
</strong>

Fail

Custom govuk-tag--red-new — a softer red background with dark red text, plus a cross icon rendered via a ::before pseudo-element pointing at /public/images/icon-tag-remove-cross.svg. No inline svg is required.

Example
Failed 
/* application.scss */
.govuk-tag--red-new {
  color: rgb(42.4, 10.6, 5.6);
  background-color: rgb(244.25, 204.5, 198.25);
}
.govuk-tag--red-new:before {
  background-image: url(/public/images/icon-tag-remove-cross.svg);
  content: "";
  display: inline-block;
  height: 10px;
  width: 10px;
  margin-left: 5px;
  margin-right: 10px;
  vertical-align: middle;
}

<!-- Usage -->
<strong class="govuk-tag govuk-tag--red-new">Failed</strong>

Yes

Same green-tick visual as Pass, used where the check answers a binary question rather than passing or failing. Used on the HEFER required? and SSSI present? checks.

Example
Yes 
<strong class="govuk-tag govuk-tag--green">
  <svg class="tick-icon" width="20" height="20" viewBox="0 0 24 24"
       fill="green" xmlns="http://www.w3.org/2000/svg">
    <path d="M9 16.2L4.8 12L3.4 13.4L9 19L21 7L19.6 5.6L9 16.2Z"/>

</strong>

No

Same red-cross visual as Fail, with the label swapped to No for binary-question checks. Currently not in active use (every HEFER required? and SSSI present? check in the prototype answers Yes) but documented here so the negative variant is on hand if it's needed.

Example
No 
<strong class="govuk-tag govuk-tag--red-new">No</strong>

All four side by side

Outcome Tag When used
Pass Passed Regulatory pass/fail checks — Available area, Moorland
Fail Failed Same checks, when the threshold is not met
Yes Yes Binary-question checks — HEFER required?, SSSI present?
No No Same binary-question checks, when the answer is No

Layout inside the check summary

When used inside a <summary> row, the application-level rule on .govuk-details__summary-text floats the tag to the right edge of the row with a 10px gap — see the Details pattern for the layout rule. No per-instance margin classes are needed.

When to use

  • To show the outcome of a single, atomic rule check at the end of its summary row.
  • To pair with the Yes/No vocabulary when the check is informational rather than pass/fail.

When not to use

  • For overall case status (use the colour tags documented on the Status tags page).
  • For task-list status indicators (use the standard Task list tags).
  • To convey severity beyond a binary outcome — Pass/Fail/Yes/No carry exactly two states; if a check has a third "in progress" or "warning" state, pick a different colour tag.

Where does this occur

  • Real-case calculations (Golden Grange) — uses all four variants (Pass + Fail on the Available area / Moorland checks, Yes on every HEFER and SSSI check).
  • Large calculations (101 parcels)
  • Medium calculations
  • Small calculations
