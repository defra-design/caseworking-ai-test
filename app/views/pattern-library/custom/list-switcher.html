List switcher

A small control that swaps the subject of a list: a label, a single-select accessible autocomplete, and a "Change" button. Used at the top of a caselist tab to view a different person's or team's cases.

On the Grasslands caselist the "My cases" tab carries a Change assignee switcher (its range is every caseworker who has cases) and the "Open cases" tab carries a Change team switcher (Team A/B/C/All teams). Picking a value and choosing Change reloads the tab for that subject and re-labels the tab to match (e.g. "R Singh's cases", "Team B cases"). It is single-select — exactly one subject is shown at a time.

Example

Example

The <select> is progressively enhanced into a type-ahead autocomplete.

<form method="get" action="caselist">
  <label class="govuk-label govuk-label--s" for="myAssignee-input">Change assignee</label>
  <select class="govuk-select" id="myAssignee-input" name="myAssignee">
    <!-- one <option> per subject (e.g. every caseworker with cases) -->
  </select>
  <button class="govuk-button govuk-button--secondary">Change</button>
</form>

<!-- progressive enhancement: single-select autocomplete -->
<script>
  accessibleAutocomplete.enhanceSelectElement({
    selectElement: document.getElementById('myAssignee-input'),
    defaultValue: 'M Walker', showAllValues: true
  })
</script>

When to use

  • When a list is normally shown for one subject (my cases, my team) but a user occasionally needs to view it for another.
  • When the set of subjects is more than a handful — the autocomplete lets users type to find one.

When not to use

  • To narrow a list by attributes (status, date) — that's a filter, not a switcher.
  • When users need to compare or combine several subjects at once — a switcher is single-select by design.

How it works

  • It is a plain <select> + submit button, so it works without JavaScript; the select is enhanced into a single-select autocomplete (see Accessible autocomplete).
  • Choosing Change submits the value, the route records it (in the session) and re-renders the list for that subject.
  • The tab/heading label reflects the current subject, so it's always clear whose list is shown.

Where does this occur

  • Grasslands — My cases ("Change assignee" — every caseworker with cases)
  • Grasslands — Open cases ("Change team" — Team A/B/C/All teams)
