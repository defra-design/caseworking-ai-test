Changed-value highlight

A paler yellow background applied to a single value (or table cell) to flag that it has changed since the previous run.

Used inside the calculations pages to draw the caseworker's eye to the specific number that has moved between two rule-check runs — for example a recalculated available area, a moorland percentage that has shifted, or an applied-for figure that no longer matches available. Pairs naturally with the calculations filter's "Calculations with changes" view: the filter narrows the page down to checks that have changed, the highlight then narrows the eye down to the value within each check.

Inline value variant

Wrap any inline number or short phrase in a <span class="value-changed"> to highlight it inside flowing text. Use this in body paragraphs (Pass / Fail reason text) and inside the "Available area calculation" explanation lines.

Example

The total available area (165.0000 ha) does not match the area applied for (169.8586 ha). This can happen when land parcel data is updated after an application has been submitted.

This parcel has a 47% intersection with the moorland layer. The target is 51%.

<p class="govuk-body">
  The total available area (<span class="value-changed">165.0000 ha</span>)
  does not match the area applied for (169.8586 ha).
</p>

Table row variant

Add the govuk-table__row--changed modifier to a <tr> to highlight the data cell only — the row header (<th>) stays unstyled so the field name remains scannable.

Example
Available area 169.8586 ha
Applied for 165.0000 ha
Action code CMOR1 
<tr class="govuk-table__row govuk-table__row--changed">
  <th scope="row" class="govuk-table__header">Applied for</th>
  <td class="govuk-table__cell">165.0000 ha</td>
</tr>

Colour choice

The background colour is #ffee80 — the GOV.UK focus yellow #ffdd00 mixed 50% toward white. It is intentionally paler than the focus colour so highlighted values do not get mistaken for an interactive focus state, while still standing out against the white page background.

/* application.scss */
.govuk-table__row--changed > td,
.value-changed {
  background-color: #ffee80;
}

When to use

  • To mark the specific value within a check that has moved between the previous and current rules-check run.
  • Inside a check whose tag already shows Failed or whose <details> is filed under "Calculations with changes" — the highlight points at why the check is in that state.
  • Multiple values within a single check can be highlighted simultaneously (e.g. both Available area and Maximum available area when both reflect the same recalculation).

When not to use

  • To indicate validation errors — use the standard warning text / error message pattern instead.
  • To highlight the user's current focus or interaction — that is the browser's job, and a near-yellow background would be confused with the focus state.
  • On every row of a table — at that point the highlight stops drawing the eye.

Research history

"Show me what changed between the original and amended application" is a recurring research demand (see Confirmation page). The research archive reuses this highlight in its before/after diff between consecutive captures of a screen — the demand and the pattern point at each other.

Where does this occur

  • Real-case calculations (Golden Grange, latest run) — inline-value variant inside Fail reasons and the Available area calculation paragraphs.
  • Real-case calculations — middle run
  • Large calculations (101 parcels, latest)
  • Large calculations — middle run
  • Medium calculations
