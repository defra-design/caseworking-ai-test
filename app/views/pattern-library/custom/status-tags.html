Status tags

Two custom tag colour variants extend the standard GOV.UK tag palette to meet specific status display needs in this prototype.

These are defined in application.scss and should only be used in the same contexts they currently appear — case status and task status indicators. Do not apply them to new contexts without checking existing usage first.

Neutral tag

.govuk-tag--neutral — white background with black text. Used for statuses that require no colour signal, such as a task that is incomplete but not yet overdue.

Example
Incomplete 
/* application.scss */
.govuk-tag--neutral {
  color: black;
  background-color: white;
}

<!-- Usage -->
<strong class="govuk-tag govuk-tag--neutral">Incomplete</strong>

Red status tag

.govuk-tag--red-status — light red background with dark red text. Used for statuses such as "Not accepted", "Cannot accept" or "There's a problem" where a softer red is more appropriate than the standard govuk-tag--red.

This is the same soft red as the Fail check-result tag — they share their colour values via the tokens in _tags.scss. What differs is vocabulary, context and one piece of styling: status tags describe the whole case ("Not accepted") and have no icon; check-result tags describe a single rules check ("Failed") and add a cross icon. Use this class for case status; use the check-result tag inside the calculations page.

Example
Not accepted   There's a problem 
/* application.scss */
.govuk-tag--red-status {
  color: rgb(42.4, 10.6, 5.6);
  background-color: rgb(244.25, 204.5, 198.25);
}

<!-- Usage -->
<strong class="govuk-tag govuk-tag--red-status">Not accepted</strong>

Where does this occur

Status tags carry case and task status across the prototype:

  • Caselist — case status against each application.
  • Task list — status against each task.
  • Agreement — agreement status.
  • Calculations — the same soft red appears (as govuk-tag--red-new) on the Fail check-result tag.
