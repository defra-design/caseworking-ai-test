Senior manager caselist Temporary

A read-focused, per-scheme variant of the team caselist aimed at a senior manager rather than a caseworker: one scheme, one "All cases" list, a funds-allocation summary and the money on each case — over the same case data.

! Warning This is a temporary / trial variant. Two of its changes — the wider layout and the Value (£K) column — are candidates to be folded into the main team caselist soon. Documented here so the trial is visible; expect it to move or be absorbed.

Where the team caselist is a caseworker's queue (my work / my team / everyone, active vs completed), this variant re-frames the same list for oversight of a single funding scheme. It keeps the data, search, Status filter, sortable table, multi-assign and pagination of the team caselist, and changes five things:

  • a single "All cases" tab instead of the three-tab My / context / Completed switcher;
  • a funds-allocation bar at the top, scoped to the scheme;
  • a Scheme metadata summary (four-column label/value table) beneath the bar;
  • a sortable Value (£K) column in the table, after Submitted; and
  • a wider page (1166px) so the extra column has room.

When to use this

  • An oversight / management view of a single scheme, where the money and the budget position matter as much as the individual cases.
  • When you want the case list and its financial summary on one screen.

When not to use this

  • A caseworker's day-to-day queue — use the team caselist (my work / team / everyone).
  • Anywhere that needs the team/context split or the active-vs-completed tabs — this variant collapses to one tab on purpose.
  • As a settled pattern — it's temporary; don't build new screens on it until the width and Value column decisions land in the main caselist.

Single "All cases" tab

The three-tab switcher is replaced by one selected "All cases" tab. The permanent SBI search, the Status "filter list" with removable filter tags, the multi-assign checkboxes and the pagination all carry over unchanged. Rendered by a single-tab schemeAllCases macro alongside the team caselist's tabsAndTable macro, both reading the same Grasslands case data.

Funds-allocation bar & scheme metadata

Above the tab sits the scheme's budget allocation bar (the split / outlined / aligned-key form) and a four-column metadata summary — two label/value pairs per row.

Value (£K) column

A sortable numeric column added after Submitted, holding each case's value in £K. It's a candidate for the main caselist. The header is kept on one line with app-th-nowrap (short numeric cells would otherwise starve the column and wrap the label), and it sorts numerically via the same server-side full-list column sorting as the other columns. Column set: Select · ID · Business · SBI · Submitted · Value (£K) · Status · Assignee.

Wider layout

The page is widened from the GOV.UK 960px page width to 1166px to give the extra column room. The class goes on the layout's width-container (via containerClasses), not an inner one — an inner container is clamped by the outer 960px wrapper. The header is widened to match (headerContainerClasses) so its left edge aligns with the table.

/* application.scss — two classes beat the GOV.UK media-query rule */
.govuk-width-container.app-caselist-wide { max-width: 1166px; }
