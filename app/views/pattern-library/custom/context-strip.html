Context strip

A light-green strip (.lg-div) below the Defra header that provides persistent case context on every case detail page.

The strip contains the back link, the applicant name as the page heading, and key case metadata — typically the application ID and a status tag, though the exact fields vary by area (see the Woodlands variant below). This keeps caseworkers oriented without needing to scroll back to the top of dense task lists.

Background colour: #E7F4EB. Do not change this to a GOV.UK colour token — the specific green tint is intentional branding.

Main case context strip

Used on all main-case casework pages. References data['caseStatus'] for the status tag.

Woodlands entitlement context strip

Used across the Woodlands entitlement flow (calculate / confirm / confirmed). This variant carries several metadata fields (Scheme, Application ID, SBI) and no status tag. The fields sit in a display:flex; flex-wrap:wrap row so they stay on one line where there's room and wrap cleanly onto a second line — never mid-value — when a long application ID (e.g. WMP-6A8-DXE) makes them too wide. The scheme label is fixed because Woodlands is a single-scheme area.

Example
Back to caselist

Mellor & Sons

Scheme: Woodland Management Plan (PA3)

Application ID: WMP-6A8-DXE

SBI: 113593357

<div class="lg-div">
  <div class="govuk-width-container">
    <a href="/Woodlands/caselist" class="govuk-back-link">Back to caselist</a>

    <div style="display:flex; flex-wrap:wrap; column-gap:2.5em;">
      <p class="govuk-body"><strong>Scheme:</strong> Woodland Management Plan (PA3)</p>
      <p class="govuk-body"><strong>Application ID:</strong> {{ "{{ c.id }}" }}</p>
      <p class="govuk-body"><strong>SBI:</strong> {{ "{{ c.sbi }}" }}</p>
    </div>
  </div>
</div>
