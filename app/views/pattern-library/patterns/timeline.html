Case history timeline Experimental

A reverse-chronological record of everything that has happened on a case — status changes, decisions, messages and notes — so any caseworker can pick it up and understand its history.

Scaffold — complete from the FRPS-D2 timeline using the MOJ timeline component.

Variant — change-history (oldest first)

Case timelines run newest first. The research archive reuses the same moj-timeline component but runs it oldest → newest, so a screen's evolution flows top-to-bottom and the "what changed" note sits in the gap before each item, ending on a green Current design — Live now terminal node. This is a documented ordering variant of the same component, not a fork — no new classes are introduced.
